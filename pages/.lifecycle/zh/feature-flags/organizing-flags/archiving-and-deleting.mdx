# 存档和删除

## 概述

本主题介绍如何存档不再需要的 feature flags，或删除已存档的 feature flags。

当一个 feature flag 不再向用户提供服务时，可能是时候将其存档了。存档一个 feature flag 会将其从 FeatBit 中停用，而不会删除它。如果需要，您可以还原一个已存档的 feature flag，或者查找它以参考其配置。

在确定项目中不再需要一个已存档的 feature flag 后，您可以将其删除。

## 存档 feature flags

当一个 feature flag 在您选择的环境中**仅提供一种变体**，并且**在过去 7 天内未被请求**时，它是存档的好候选。

在 FeatBit 中存档一个 feature flag 时，您也应该从代码中移除该 feature flag。

<Callout type="warning">如果您不从代码中移除一个已存档的 feature flag，那么所有遇到该功能的终端用户都将接收到回退值。</Callout>

<Callout type="error">在存档一个 feature flag 之前，请确保这样做不会产生任何意外的影响。</Callout>

要存档一个 feature flag：

1. 导航到 feature flags 列表。
2. 找到您想要存档的 feature flag。
3. 点击 feature flag 名称或“详情”链接，前往 feature flag 详情页面。
4.  点击 **存档**。一个确认对话框会出现。

![](../../feature-flags/assets/organizing-flags/archiving-and-deleting/001.png)

5. 点击 **确定**。

该 feature flag 被存档并从 feature flags 列表中移除。您可以在已存档的 feature flags 列表中找到它。要了解更多信息，请阅读查看和恢复已存档的 feature flags。

## 查看已存档的 feature flags

已存档的 feature flags 在 feature flags 列表中是不可见的。勾选“已存档”复选框以查看它们：

![](../../feature-flags/assets/organizing-flags/archiving-and-deleting/002.png)

## 恢复和删除已存档的 feature flags

存档一个 feature flag 后，您可以恢复或删除它。

当您恢复一个已存档的 feature flag 时，它会像任何其他 feature flag 一样重新开始评估。因此，在恢复之前，重要的是要确认该 feature flag 不会产生任何意外的影响。

<Callout type="warning">当您删除一个已存档的 feature flag 时，所有用户将会接收到代码中定义的 feature flag 值。在您删除它之前，请从应用程序代码中移除所有对该 feature flag 的引用。</Callout>

<Callout type="error">如果您删除了一个 feature flag，您将无法恢复它。在您删除一个 feature flag 之前，请确保您不再需要它。</Callout>

要恢复或删除一个 feature flag：

1. 导航到 feature flags 列表。
2. 勾选“已存档”复选框以查看已存档的 feature flags。
3. 找到您想要恢复或删除的 feature flag。
4. 点击 feature flag 名称或“详情”链接，前往 feature flag 详情页面。
5.  如果您想要恢复这个 feature flag，请点击 **恢复**；如果您想要删除这个 feature flag，请点击 **永久删除**。
