# 创建特性标志变体

### 概览

本主题阐释了如何创建和编辑您的特性标志变体，并概述了标志变体的不同类型。

### 了解标志类型

FeatBit 支持布尔型和多变量标志。在**变体**部分，您可以添加、编辑或删除现有标志的变体：

* **布尔**型标志有两种变体：`true` 或 `false`。
* **多变量**标志可以有两种以上的变体。允许的变体取决于标志的类型。多变量标志的类型包括字符串、数字和 JSON。

在**变体**部分，您可以更改变体类型，添加、编辑或删除现有标志的变体。

#### 删除变体

当您删除一个变体时，如果该变体被任何规则使用，或者一些特定用户已被加入到该变体中，系统会弹出一个警告信息，您必须在移除该变体之前修改这些规则。

### 更改默认的标志值

当您创建一个特性标志时，一些值会被设定为默认值。除非您另有指定，否则标志的默认值是在标志**开启**或**关闭**时提供的值。标志**开启**时的默认值可以是单一变体或分批发布。标志**关闭**时的默认值只能是单一变体。

例如，一个布尔型标志可以在开启时将 true 设定为默认规则，在关闭时将 false 设定为默认变体。一个多变量标志可以在开启时将变体 2 设定为默认规则，在关闭时将变体 1 设定为默认变体，并配置变体 3 和变体 4 在用户符合特定定位规则时显示。

每次您创建一个特性标志时，系统都会自动指定默认值。您可以接受这些默认值或者进行更改。

您可以通过更改**如果关闭，则提供**或修改默认规则的提供值来更改标志的默认值。

![](../feature-flags/assets/create-flag-variations/002.webp)

### 了解多变量标志

多变量标志允许您使用一个标志同时提供多于两种变体的功能。您可以为多变量标志添加的变体数量没有限制，这对于复杂的用例以及管理功能的多个变体非常有用。

多变量标志的变体类型包括字符串、数字或 JSON。

您可以使用百分比发布来细化功能的推出。例如，一个提供字符串变体的多变量标志可以用于为恐龙游戏启用不同难度级别，通过百分比发布向用户提供这些难度。在此示例中，默认规则为 10% 的用户提供高难度，为 10% 的用户提供低难度，为剩余 80% 的用户提供中等难度。

以下是一个带有百分比发布的标志图片：

![](../feature-flags/assets/create-flag-variations/003.webp)

您可以使用自定义规则来针对多组用户并进一步细化发布。在下一个示例中，第一条规则为所有年龄小于 20 岁的用户提供高难度。第二条规则针对国籍为 FR 或 IT 的用户，为这些用户中的 10% 提供高难度，10% 提供低难度，80% 提供中等难度。所有未被自定义规则定位的用户都将接收默认规则。

以下是一个带有自定义规则的标志图片：

![](../feature-flags/assets/create-flag-variations/004.webp)

您也可以使用多变量标志来执行测试，以使用 FeatBit 的实验功能。要了解更多信息，请阅读[创建实验](../experimentation/creating-experiments.md)。
