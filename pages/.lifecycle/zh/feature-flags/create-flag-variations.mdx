# 创建特性标志变体

### 概览 <a href="#overview" id="overview"></a>

本主题介绍如何创建和编辑特性标志的变体，并概述不同类型的标志变体。

### 理解标志类型 <a href="#understanding-flag-types" id="understanding-flag-types"></a>

FeatBit 支持布尔型和多变量型标志。在**变体**部分，您可以添加、编辑或删除现有标志的变体：

* **布尔型**标志有两种变体：`true` 或 `false`。
* **多变量型**标志可以有两个以上的变体。允许的变体取决于标志的类型。多变量型标志的类型有字符串、数字和 JSON。

在**变体**部分，您可以更改变体类型，添加、编辑或删除现有标志的变体。

![](../feature-flags/assets/create-flag-variations/001.webp)

#### 删除变体

当您删除一个变体时，如果该变体被任何规则使用，或者已经将某些目标用户添加到该变体中，则会出现一条警告信息，您必须在移除变体之前修改这些规则。

### 更改默认标志值 <a href="#changing-default-flag-values" id="changing-default-flag-values"></a>

创建特性标志时，会为其指定一些默认值。标志的默认值是当标志处于开启或关闭状态时提供的值，除非您指定了其他值。当标志处于**开启**状态时，默认值可以是单一变体或者是分布策略。当标志处于**关闭**状态时，默认值只能是单一变体。

例如，一个布尔型标志可能在开启时将`true`设置为其默认规则，并在关闭时将`false`设置为其默认变体。一个多变量型标志可能在开启时将`variation 2`设置为其默认规则，并在关闭时将`variation 1`设置为其默认变体，同时配置`variation 3`和`variation 4`在用户匹配特定目标规则时出现。

每次创建特性标志时，都会自动指定默认值。您可以接受这些默认值或者更改它们。

您可以通过更改**如果关闭，提供**或更改默认规则的提供值来更改标志的默认值。

![](../feature-flags/assets/create-flag-variations/002.webp)

### 理解多变量型标志 <a href="#understanding-multivariate-flags" id="understanding-multivariate-flags"></a>

多变量型标志允许您使用一个标志同时提供多于两种变体的特性。您可以为多变量型标志添加的变体数量没有限制，这使其适用于复杂的使用场景以及管理特性的多个变体。

多变量型标志的变体类型包括字符串、数字或 JSON。

您可以使用百分比分布策略来细化特性的推出。例如，一个提供字符串变体的多变量型标志可以用来为恐龙游戏启用不同难度级别，向用户提供这些难度的百分比分布。在这个示例中，默认规则为10%的用户提供高难度，为10%的用户提供低难度，为80%的用户提供中等难度。

这是一个带有百分比分布策略的标志的图像：

![](../feature-flags/assets/create-flag-variations/003.webp)

您可以使用自定义规则来针对多个用户群体，并进一步细化推出策略。在下一个示例中，第一条规则为所有年龄小于20岁的用户提供高难度。第二条规则针对国家为FR或IT的用户，为其中10%的用户提供高难度，10%的用户提供低难度，80%的用户提供中等难度。所有未被自定义规则涵盖的用户都将接收默认规则。

这是一个带有自定义规则的标志的图像：

![](../feature-flags/assets/create-flag-variations/004.webp)

您还可以使用多变量型标志通过FeatBit的实验功能进行测试。要了解更多信息，请阅读[创建实验](../experimentation/creating-experiments.md)。
