# 3. 连接 SDK

使用 SDK 是理解如何通过功能标志交付、控制和试验软件的基本技能。

在上一步“尝试与演示交互”中，您已经学会了如何使用功能标志使游戏组件出现/消失，并切换游戏的难度模式。现在让我们看看它在代码中是如何实现的。

## 按照门户中的指南进行操作

最简单的方法是按照门户“入门”页面上的说明进行操作。在门户中，您可以：

**步骤 1**。选择刚刚在上一步中创建的功能标志之一。然后单击“下一步”按钮，转到步骤 2。

![](../getting-started/assets/connect-to-sdk/001.png)

**步骤 2**。选择一种您熟悉的编程语言。复制自动生成的代码，并在您的集成开发环境中运行它。

![](../getting-started/assets/connect-to-sdk/002.png)

门户生成的代码包含了所有必要的元素，例如：

- SDK 安装
- 使用服务器 URL、环境密钥进行 SDK 初始化
- 使用自定义属性标识用户
- 功能标志实施
- 功能标志评估
- 功能标志更新监听器
- 等等

![](../getting-started/assets/connect-to-sdk/003.png)


## 手动连接 SDK

我们提供了多种客户端、服务器端和移动 SDK 供您选择。您可以选择您擅长的语言，以加快完成快速入门。

<Callout type="info" emoji="👾">
  我们建议您使用我们的 SDK，因为我们已经为您实施了最佳实践。
</Callout>

### Javascript

我们提供了客户端 [Javascript/Typescript SDK](https://github.com/featbit/featbit-js-client-sdk)。我们还为 [React](https://github.com/featbit/featbit-samples/tree/main/samples/dino-game/interactive-demo-react) 和 [Vue](https://github.com/featbit/featbit-samples/tree/main/samples/dino-game/interactive-demo-vue) 提供了“恐龙游戏”示例。此 SDK 可与 Typescript、React、Vue、Angular 等完美配合。

该 SDK 的 GitHub 存储库包含了如何使用它的完整教程。它还提供了一个示例项目供您尝试。

### .NET

我们提供了 [服务器端 .NET SDK](https://github.com/featbit/featbit-dotnet-sdk) 供您使用。此 SDK 可用于您的控制台应用程序或 ASP.NET Core 应用程序。

它还可以用于前端应用程序，如 WPF、WinForm、MAUI 等。但我们不建议您在移动应用程序中使用它，因为它比真正的移动 SDK 消耗更多的电池电量。

该 SDK 的 GitHub 存储库包含了如何使用它的完整教程。它还提供了一个示例项目供您尝试。

### Java

我们提供了 [Java SDK](https://github.com/featbit/featbit-java-sdk) 供您使用。此 SDK 可用于您的控制台应用程序或 Spring Boot 应用程序。

该 SDK 的 GitHub 存储库包含了如何使用它的完整教程。它还提供了一个示例项目供您尝试。

### Python

我们提供了 [Python SDK](https://github.com/featbit/featbit-python-sdk)。该 SDK 的 GitHub 存储库包含了如何使用它的完整教程。它还提供了一个示例项目供您尝试。

### Go

我们提供了 [Go SDK](https://github.com/featbit/featbit-go-sdk)。该 SDK 的 GitHub 存储库包含了如何使用它的完整教程。它还提供了一个示例项目供您尝试。


### REST API 和 WebSocket

对于我们尚未提供 SDK 的任何其他语言或框架，您可以使用我们的 REST API 来获取功能标志。请参阅此处的文档[](../api-docs/get-users-feature-flags-with-api)。 

您可以使用我们的 WebSocket 接口来获取实时功能标志更新。请加入我们的 Slack 频道以获取有关此功能的更多信息。
