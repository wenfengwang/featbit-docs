# 使用 Docker Compose 进行安装

## 先决条件

* Docker 引擎版本 **1.13.1+**
* Docker Compose **1.27.0+**

## 安装

FeatBit 提供两种计划：**标准版** 和 **专业版**。适合您的计划将取决于您的业务规模和您的具体需求。一般来说，标准版计划非常适合小型企业，而专业版计划则非常适合中大型企业。您可以在此处找到有关它们的差异的更多信息[这里](../tech-stack/standard-vs.-professional.md)。所有计划都保证了弹性、速度、访问我们 Slack 社区的权限以及我们团队的支持。

*   将存储库克隆到您的服务器或本地计算机，然后启动服务。

    ```bash
    git clone https://github.com/featbit/featbit
    cd featbit

    # 对于标准版，请运行
    docker compose up -d

    # 对于专业版，请运行
    docker compose -f docker-compose-pro.yml up -d
    ```
* 所有容器启动后，转到 FeatBit 的门户 [http://localhost:8081](http://localhost:8081/) 并使用默认凭据登录。
  * 用户名： **test@featbit.com**
  * 密码： **123456**

如果您想要从源代码构建映像并启动容器，也可以按以下步骤操作：&#x20;

```sh
# 对于标准版，请运行
docker compose up -f docker-compose-dev.yml -d

# 对于专业版，请运行
docker compose -f docker-compose-pro-dev.yml up -d
```

<提示框 type="info">
专业版的 docker compose 文件包含一个名为 **init-kafka-topics** 的“运行一次”服务，用于创建系统所需的 kafka 主题。此服务仅运行一次，然后退出，因此此服务处于已退出状态是可以接受的。
</提示框>

## 更新

运行以下命令以更新到最新的 FeatBit 映像：

```bash
docker-compose pull
docker-compose rm -fsv featbit
docker-compose up -d
```
