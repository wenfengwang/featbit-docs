# 针对用户进行标记

## 概述 <a href="#overview" id="overview"></a>

此类别说明如何使用标记的 **目标定位** 标签来控制哪些用户接收特定功能标记的变种。配置哪些用户接收哪些标记变种被称为标记的目标定位。

## 使用目标定位标签 <a href="#using-the-targeting-tab" id="using-the-targeting-tab"></a>

您可以使用 **目标定位** 标签来在进行更广泛的发布之前，为内部测试、私人测试版或可用性测试滚动功能。如果您知道只想让用户在特定时间段内收到标记，甚至可以为标记的目标定位设置过期日期（未来功能）。

尽管 **目标定位** 标签使用定位的 "用户" 语言，但用户可以是任何唯一对应于目标的标识符。您可以将应用程序的用户、电子邮件地址、系统、服务、机器、资源或任何其他可以唯一标识的内容作为目标。
