# 创建标志变量

### 概览 <a href="#overview" id="overview"></a>

本主题解释了如何创建和编辑特性标志的变量，并概述了不同类型的标志变量。

### 了解标志类型 <a href="#understanding-flag-types" id="understanding-flag-types"></a>

FeatBit支持布尔和多元变量标志。在**变量**部分，您可以添加、编辑或删除现有标志的变量：

* **布尔**标志有两种变量：`true`或`false`。
* **多元**标志可以有多于两种的变量。允许的变量取决于标志的类型。多元标志类型是字符串、数字和JSON。

在**变量**部分，您可以更改变量类型，添加、编辑或删除现有标志的变量。

这是一个布尔变量的示例

![](../feature-flags/assets/create-flag-variations/001.webp)

#### 删除变量

当您删除一个变量时，如果该变量被任何规则使用或某些目标用户已被添加到该变量中，会出现警告消息，您必须在删除变量之前修改这些规则。

### 更改默认的标志值 <a href="#changing-default-flag-values" id="changing-default-flag-values"></a>

当您创建一个特性标志时，它的一些值被指定为默认值。标志的默认值是在标志打开或关闭时提供的值，除非您另外指定。当标志为**打开**时的默认值可以是单个变量或分段。当标志为**关闭**时的默认值只能是单个变量。

例如，一个布尔标志可以在打开时将`true`设置为其默认规则，将`false`设置为其默认关闭变量。一个多元标志可以在打开时将`variation 2`设置为其默认规则，将`variation 1`设置为其默认关闭变量，将`variation 3`和`variation 4`配置为在用户匹配某些目标规则时显示。

默认值在每次创建特性标志时都会自动指定。您可以接受默认值或更改它们。

您可以通过更改**如果关闭，则提供**来更改标志的默认值，或者更改默认规则的提供值。

![](../feature-flags/assets/create-flag-variations/002.webp)

### 了解多元变量标志 <a href="#understanding-multivariate-flags" id="understanding-multivariate-flags"></a>

多元变量标志允许您使用一个标志同时提供两个以上的特性变量。您可以为多元标志添加无限数量的变量，这对于复杂的用例和管理多个特性变量非常有用。

多元变量标志的变量类型包括字符串、数字或JSON。

您可以使用百分比分段来调整特性的分段。例如，一个以字符串作为变量的多元变量标志可以用于为恐龙游戏启用不同的难度，向用户提供这些类型的百分比分段。在这个例子中，当有10%的用户时，默认规则为困难难度，10%的用户为简单难度，80%的用户为中等难度。

这是一个带有百分比分段的标志的图像：

![](../feature-flags/assets/create-flag-variations/003.webp)

您还可以使用自定义规则来针对多个用户集并进一步调整分段。在下一个示例中，第一条规则为年龄小于20岁的所有用户提供困难难度。第二条规则针对国家为FR或IT的用户，并为这些用户的10%提供困难难度，10%提供简单难度，80%提供中等难度。在自定义规则中未被定位的所有用户将提供默认规则。

这是一个带有自定义规则的标志的图像：

![](../feature-flags/assets/create-flag-variations/004.webp)

您还可以使用多元变量标志来执行FeatBit的实验功能的测试。要了解更多，请阅读[创建实验](../experimentation/creating-experiments.md)。
