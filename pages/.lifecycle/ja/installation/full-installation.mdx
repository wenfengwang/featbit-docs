# Docker Composeを使用したインストール

## 前提条件

* Docker Engine バージョン **1.13.1+**
* Docker Compose **1.27.0+**

## インストール

FeatBitには**Standard**と**Professional**の2つのプランがあります。あなたに適したプランは、ビジネスの規模や特定のニーズによって異なります。一般的に、Standardプランは中小企業向きであり、Professionalプランは中規模および大規模企業向きです。それぞれの違いについては[こちら](../tech-stack/standard-vs.-professional.md)で詳細をご覧いただけます。全てのプランには、耐障害性、スピード、当社のSlackコミュニティへのアクセス、および当社チームからのサポートが保証されています。

*   リポジトリをサーバーまたはローカルマシンにクローンし、サービスを起動します。

    ```bash
    git clone https://github.com/featbit/featbit
    cd featbit

    # Standard版の場合、次のコマンドを実行します
    docker compose up -d

    # Professional版の場合、次のコマンドを実行します
    docker compose -f docker-compose-pro.yml up -d
    ```
* すべてのコンテナが起動したら、FeatBitのポータル[http://localhost:8081](http://localhost:8081/)にアクセスし、デフォルトの資格情報を使用してログインします。
  * ユーザー名: **test@featbit.com**
  * パスワード: **123456**

ソースコードからイメージをビルドし、コンテナを起動する場合は、以下のようにします:&#x20;

```sh
# Standard版の場合、次のコマンドを実行します
docker compose up -f docker-compose-dev.yml -d

# Professional版の場合、次のコマンドを実行します
docker compose -f docker-compose-pro-dev.yml up -d
```

<Callout type="info">
Pro版のdocker composeファイルには、システムで必要なkafkaトピックを作成する "一度だけ" の**init-kafka-topics**サービスが含まれています。このサービスは一度だけ実行され、その後は終了することを意図しているため、このサービスがExited状態であることは問題ありません。
</Callout>

## アップデート

最新のFeatBitイメージにアップデートするには、以下のコマンドを実行してください:

```bash
docker-compose pull
docker-compose rm -fsv featbit
docker-compose up -d
```
