# Docker Composeを使用したインストール

## 必要条件

* Docker Engineバージョン **1.13.1+**
* Docker Compose **1.27.0+**

## インストール

FeatBitには2つのプランがあります: **Standard**と**Professional**。適切なプランは、ビジネスの規模や特定のニーズによって異なります。一般的に、Standardプランは中小企業に最適であり、Professionalプランは中大企業に最適です。プランの違いについては[こちら](../tech-stack/standard-vs.-professional.md)をご覧ください。すべてのプランは、耐久性、速度、Slackコミュニティへのアクセス、チームからのサポートを保証しています。

*   リポジトリをサーバーまたはローカルマシンにクローンし、サービスを起動します。

    ```bash
    git clone https://github.com/featbit/featbit
    cd featbit

    # Standardバージョンの場合は、次のコマンドを実行します
    docker compose up -d

    # Professionalバージョンの場合は、次のコマンドを実行します
    docker compose -f docker-compose-pro.yml up -d
    ```
* すべてのコンテナが起動したら、FeatBitのポータル [http://localhost:8081](http://localhost:8081/) に移動し、デフォルトの資格情報を使用してログインしてください。
  * ユーザー名: **test@featbit.com**
  * パスワード: **123456**

ソースコードからイメージをビルドし、コンテナを起動したい場合は、以下の手順を実行することもできます:&#x20;

```sh
# Standardバージョンの場合は、次のコマンドを実行します
docker compose up -f docker-compose-dev.yml -d

# Professionalバージョンの場合は、次のコマンドを実行します
docker compose -f docker-compose-pro-dev.yml up -d
```

<Callout type="info">
プロバージョンのdocker composeファイルには、システムで必要なkafkaのトピックを作成するための「一度だけ実行する」サービスである **init-kafka-topics** が含まれています。このサービスは1回だけ実行して終了することを意図しているため、Exitedステータスであることは問題ありません。
</Callout>

## アップデート

最新のFeatBitイメージにアップデートするには、次のコマンドを実行します:

```bash
docker-compose pull
docker-compose rm -fsv featbit
docker-compose up -d
```
