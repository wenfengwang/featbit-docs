# APIでフィーチャーフラグを取得する

このドキュメントでは、ターゲットSDKが利用できない場合に、サーバーから直接フィーチャーフラグを取得する包括的なガイドを提供します。

評価サーバーでは、クライアント側およびサーバー側プラットフォーム向けに、希望する環境に特化したフィーチャーフラグを簡単に取得できる2つのAPIが提供されています。

クライアント側とサーバー側のSDKの違いは次のとおりです。

* クライアント側SDKは単一のユーザー向けですが、サーバー側SDKは複数のユーザー向けです。
* クライアント側SDKはすべてのフラグを評価するために評価サーバーに依存しますが、サーバー側SDKはフラグをローカルで評価します。

<Callout type="info" emoji="ℹ️">
APIリクエストの`{Host}`は、環境に適したホストURLに置き換える必要があることに注意してください。`{Host}`には評価サーバーホストまたはFeatBitエージェントホストを利用できますが、両方とも同じAPIを提供します。
</Callout>

## クライアント側

クライアント側リクエストでは、`Authorization`ヘッダーにクライアント環境シークレットを指定します。

**リクエスト**

```bash
curl -X POST {Host}/api/public/sdk/client/latest-all \
-H 'Content-Type: application/json' \
-H 'Authorization: client-env-secret' \
-d '{
    "keyId": "bot-id",
    "name": "bot",
    "customizedProperties": [
        {
            "name": "level",
            "value": "high"
        },
        {
            "name": "localtion",
            "value": "us"
        }
    ]
}'
```

**レスポンス**

```json
[
    {
       "id":"hello-world",
       "variation":"true",
       "variationType":"boolean"
    }
]
```

## サーバー側

サーバー側リクエストでは、`Authorization`ヘッダーにサーバー環境シークレットを指定します。

**リクエスト**

```bash
curl -H "Authorization: server-env-secret" {Host}/api/public/sdk/server/latest-all
```
