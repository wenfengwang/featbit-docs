# 個別のユーザーターゲティング

## 概要 <a href="#overview" id="overview"></a>

このトピックでは、フラグの**ターゲティング**タブを使用して、個別のユーザーがフラグのバリエーションを受け取るかどうかを制御する方法について説明します。

## ユーザーをバリエーションに割り当てる <a href="#assigning-users-to-a-variation" id="assigning-users-to-a-variation"></a>

**ターゲティング**タブの「ターゲティングユーザー」セクションでは、個別のユーザーを特定のフラグのバリエーションに割り当てることができます。

> 個別にターゲットを絞るのは少数のユーザーに対して
>
> 個々に100人以上のユーザーをターゲットにすることはパフォーマンスの低下を引き起こす可能性があるため、非常に少数の個別のユーザーに対して個別のターゲティングを使用することをお勧めします。

アプリケーションが既にデータをFeatBitに送信している場合、名前またはユーザーキーでユーザーを検索することができます。これらの文字列は大文字と小文字を区別します。ユーザーの行の上にカーソルを置いて、そのユーザーに関するより多くの属性を表示します。

FeatBitが知らないユーザーをターゲットにする必要がある場合は、そのキーを手動で入力することができます。これらのユーザーは、フィーチャーフラグに遭遇するまで黄色で表示されます。

以下のスクリーンショットでは、特定のユーザーがフラグの最初のバリエーションを受け取っています。そのフラグは彼らにターゲットされています。これは、これらのユーザーがそのフラグにアクセスしたときにそのフラグのバリエーションを受け取ることを意味します。

## ターゲットされたユーザーの削除 <a href="#removing-targeted-users" id="removing-targeted-users"></a>

ユーザーをフラグのターゲティングリストから削除するには、ユーザーの名前をクリックします。以下の手順で行います：

1. フラグの**ターゲティング**タブに移動します。
2. 「ターゲティングユーザー」セクションで、削除したいユーザーの名前を見つけます。
3. 名前の横の**X**をクリックして、彼らを削除します：
