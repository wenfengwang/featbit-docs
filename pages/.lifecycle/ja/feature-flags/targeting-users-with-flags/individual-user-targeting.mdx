# 個別ユーザーのターゲティング

## 概要 <a href="#overview" id="overview"></a>

このトピックでは、フラグの**ターゲティング**タブを使用して、個々のユーザーに機能フラグのバリエーションを受け取るかどうかを制御する方法について説明します。

## バリエーションへのユーザーの割り当て <a href="#assigning-users-to-a-variation" id="assigning-users-to-a-variation"></a>

**ターゲティング**タブの「ユーザーのターゲティング」セクションでは、特定のユーザーを特定のフラグのバリエーションに割り当てることができます。

> 個々のユーザーを対象にする場合は、少数のユーザーにのみ対象を絞ることをお勧めします。
>
> 1つのバリエーションに対して100人以上の個々のユーザーを対象にすると、パフォーマンスの低下が発生する可能性があります。

すでにアプリケーションがFeatBitにデータを送信している場合は、名前やユーザーキーでユーザーを検索できます。これらの文字列は大文字と小文字を区別します。ユーザー行の上にカーソルを合わせると、そのユーザーの詳細属性が表示されます。

FeatBitが把握していないユーザーを対象にする必要がある場合は、キーを手動で入力することができます。これらのユーザーは、フィーチャーフラグに遭遇するまで黄色で表示されます。

以下のスクリーンショットでは、特定のユーザーがフラグの最初のバリエーションを受け取っています。そのフラグは彼らをターゲットにしています。これは、これらのユーザーがそのフラグにアクセスしたときにそのフラグのバリエーションを受け取ることを意味します。

![](../../feature-flags/assets/targeting-users-with-flags/individual-user-targeting/001.webp)

## ターゲットされたユーザーの削除 <a href="#removing-targeted-users" id="removing-targeted-users"></a>

フラグのターゲットリストからユーザーを削除するには、ユーザーの名前をクリックします。手順は次の通りです。

1. フラグの**ターゲティング**タブに移動します。
2. 「ユーザーのターゲティング」セクションで、削除したいユーザーの名前を見つけます。
3. その名前の横にある**X**をクリックして削除します：

![](../../feature-flags/assets/targeting-users-with-flags/individual-user-targeting/002.webp)
