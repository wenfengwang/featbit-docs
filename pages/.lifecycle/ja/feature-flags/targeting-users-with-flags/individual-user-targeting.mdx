# 個々のユーザーをターゲットにする

## 概要 <a href="#overview" id="overview"></a>

このトピックでは、フラグの **ターゲティング** タブを使用して、機能フラグのバリエーションを受け取る個々のユーザーを制御する方法について説明します。

## バリエーションへのユーザーの割り当て <a href="#assigning-users-to-a-variation" id="assigning-users-to-a-variation"></a>

**ターゲティング** タブの「ユーザーのターゲティング」セクションでは、個々のユーザーを特定のフラグバリエーションに割り当てることができます。

> 個別にターゲットするユーザーは少数にしてください
>
> ごく少数の個人ユーザーに対してのみ個別ターゲティングを使用することを推奨します。バリエーションごとに100人以上のユーザーを個別にターゲットすると、パフォーマンスが低下する可能性があります。

アプリケーションが既にFeatBitにデータを送信している場合、名前やユーザーキーでユーザーを検索できます。これらの文字列は大文字と小文字を区別します。ユーザー行にマウスを合わせると、そのユーザーの詳細属性が表示されます。

FeatBitがまだ認識していないユーザーをターゲットにする必要がある場合、そのユーザーキーを手動で入力できます。これらのユーザーは、フィーチャーフラグに遭遇するまで黄色で表示されます。

以下のスクリーンショットでは、特定のユーザーがフラグの最初のバリエーションを受け取っています。このフラグは彼らにターゲットされています。つまり、これらのユーザーはフラグにアクセスすると、そのフラグバリエーションを受け取ります。

![](../../feature-flags/assets/targeting-users-with-flags/individual-user-targeting/001.webp)

## 対象ユーザーの削除 <a href="#removing-targeted-users" id="removing-targeted-users"></a>

フラグのターゲティングリストからユーザーを削除するには、そのユーザーの名前をクリックします。以下の手順に従ってください：

1. フラグの **ターゲティング** タブに移動します。
2. 「ユーザーのターゲティング」セクションで、削除したいユーザーの名前を見つけます。
3.  その名前の隣にある **X** をクリックして削除します。
