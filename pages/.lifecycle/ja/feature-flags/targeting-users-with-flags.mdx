# フラグを用いたユーザーターゲティング

## 概要 <a href="#overview" id="overview"></a>

このカテゴリーは、フラグの**ターゲティング**タブを使用して、どのユーザーが機能フラグのバリエーションを受け取るかを制御する方法について説明します。どのユーザーにどのフラグバリエーションを配布するかを設定することを、フラグのターゲティングと言います。

## ターゲティングタブの使用 <a href="#using-the-targeting-tab" id="using-the-targeting-tab"></a>

**ターゲティング**タブを使用して、内部テスト、プライベートベータ、またはユーザビリティテストのために機能を展開し、その後より広範囲なロールアウトを行うことができます。特定の期間だけユーザーにフラグを受け取らせたい場合は、フラグターゲティングの有効期限を設定することも可能です（将来の機能）。

**ターゲティング**タブでは「ユーザー」という言葉を使いますが、ユーザーはターゲットに一意に対応する任意の識別子を指すことができます。アプリケーションのユーザー、メールアドレス、システム、サービス、マシン、リソース、または一意に識別可能なその他のものをターゲットにすることができます。
