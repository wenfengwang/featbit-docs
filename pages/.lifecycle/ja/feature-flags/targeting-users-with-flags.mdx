# フラグを持つユーザーを対象にする

## 概要 <a href="#overview" id="overview"></a>

このカテゴリでは、フラグの「ターゲティング」タブを使用して、どのユーザーがフラグのバリエーションを受け取るかを制御する方法について説明します。どのユーザーがどのフラグのバリエーションを受け取るかを設定することを、フラグのターゲティングと呼びます。

## ターゲティングタブの使用方法 <a href="#using-the-targeting-tab" id="using-the-targeting-tab"></a>

**ターゲティング**タブを使用して、フィーチャーを内部テスト、プライベートベータ版、または使い勝手のテストの前にリリースすることができます。さらに、特定の期間にユーザーがフラグを受け取ることを望む場合は、フラグのターゲティングに有効期限を設定することもできます（将来の機能）。

**ターゲティング**タブは、対象となる「ユーザー」という言葉を使用していますが、ユーザーは対象に対応する固有の識別子であることが必要です。アプリケーションのユーザー、メールアドレス、システム、サービス、マシン、リソース、または他の識別可能なものを対象にすることができます。
