# 機能ワークフロー

### 概要 <a href="#overview" id="overview"></a>

このトピックでは、FeatBitで複雑なワークフローを作成し、機能フラグの使用をより細かくコントロールする方法について説明します。ワークフローは、チームがソフトウェアをビルド、出荷、およびコントロールする方法に対応しています。また、FeatBitを組織の既存のツールやプロセスに統合するカスタムリリース戦略を定義することも可能です。

### 利用可能なフィーチャー <a href="#available-features" id="available-features"></a>

機能ワークフローのユーザーが利用できるフィーチャーは以下の通りです：

* [フラグトリガー](flag-triggers)
* [スケジュールされたフラグ変更](scheduled-flag-changes)
