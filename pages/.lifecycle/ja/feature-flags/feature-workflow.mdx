# 機能のワークフロー

### 概要 <a href="#overview" id="overview"></a>

このトピックでは、FeatBit内で複雑なワークフローを作成することで、フィーチャーフラグの使用をより細かく制御する方法について説明します。ワークフローは、チームがソフトウェアをビルド、出荷、制御する方法に対応しています。また、カスタムリリース戦略を定義することもできます。これにより、FeatBitを組織の既存のツールやプロセスに統合することができます。

### 利用可能な機能 <a href="#available-features" id="available-features"></a>

以下の機能を利用することができるフィーチャーフラグワークフローのユーザーです:

* [フラグトリガー](flag-triggers)
* [スケジュールされたフラグの変更](scheduled-flag-changes)
